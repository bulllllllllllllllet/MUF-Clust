
编写的代码尽量精简、不要添加不必要的冗余逻辑
编写的每个函数前需要有中文注释表明函数的作用。
除非用户有需求，否则不要添加回退逻辑、默认参数等代码。需要用户确认后才能添加这些代码。

输出控制台信息的时候，需要使用分隔标记，使输出信息易读。例如：
```
_____
[INFO] 开始处理 sample_001.tiff
_____
```



绘图时使用英文，代码注释、终端输出的debug信息使用中文，但是不需要直接输出“中文：”这种字样，直接输出中文即可。

用 opencv 可视化每步；
用 WandB 记录和展示训练过程；
用 FiftyOne 分析错误样本；
若要展示或交互，用 Gradio / Streamlit 包成前端演示。
